DCR - Dexoron Cargo Realization

DCR is a utility for managing C/C++ projects. It defines a unified project structure, provides simple management commands, and uses a single configuration file: dcr.toml.

GitHub Documentation

Why

  • Unified project structure
  • Simple management commands
  • Easy setup and usage

Features

  • Project creation / initialization
  • Project build (multiple build profiles)
  • Dependency management (in development)

Installation

To install DCR, use your system terminal:

curl -fsSL https://dcr.dexoron.su/install.sh | bash

If the installation was successful, run the following command to test:

                    dcr --version

Commands

Below are the basic commands. All available commands can be found in the documentation.

Command Usage Description
dcr dcr Show help information
dcr new dcr new hello Creates a project named hello in the current directory
dcr init dcr init Creates a project in the current directory using the directory name
dcr build dcr build --release Builds the project. If no profile is specified, --debug is used.
dcr run dcr run --release Builds and runs the binary. If no profile is specified, --debug is used.
dcr clean dcr clean release Removes a build profile from target. If no profile is specified, removes the entire folder.

Example

This section shows an example of using DCR.

1. Creating a project

We will create a project named hello.

                        dcr new hello

The resulting structure will look like this:

                        hello/
                        - src/
                        - - main.c
                        - dcr.toml

2. Building the project

Enter the project directory:

                        cd hello

Build the project with the --release flag:

                        dcr build --release

The resulting structure will look like this:

                        hello/
                        - src/
                        - - main.c
                        - target/
                        - - release/
                        - - - main
                        - dcr.toml
                        - dcr.lock

3. Running the project

There are two ways to run the project:

First method (recommended)

                            dcr run --release

This command will build the project (if needed) and run it.

Second method

                            ./target/release/main

This runs the binary directly.

If you have questions about any of the steps, please refer to the documentation.
