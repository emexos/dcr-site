DCR - Dexoron Cargo Realization

DCR — это утилита для управления проектами на C/C++. Она задает единый формат проекта, дает команды для управления, и использует один конфигурационный файл dcr.toml

GitHub Документация

Зачем

  • Единая структура проекта
  • Простые команды для управления
  • Легкость настройки и использования

Возможности

  • Создание/Инициализация проекта
  • Сборка проекта(разные профили сборки)
  • Управление зависимостями(в разработке)

Установка

Для установки DCR используйте терминал системы:

curl -fsSL https://deno.land/install.sh | sh

Если все прошло успешно, пропишите следуюшую команду для теста:

                    dcr --version

Команды

Ниже перечислены базовые команды, все команды можно найти в документации

Команда Использование Описание
dcr dcr Показать справочную информацию
dcr new dce new hello Создает проект в текущей директории с именем hello
dcr init dcr init Создает проект в текущей директории. Имя проекта будет равно имени директории
dcr build dcr build --release Собирает проект. Если профиль не указан, используется --debug.
dcr run dcr run --release Собирает проект и запускает бинарник. Если профиль не указан, используется --debug.
dcr clean dcr clean release Удаляет профиль из target. Если не указан профиль, удаляется вся папка

Пример

Здесь показан пример использования DCR

1. Создание проекта

Мы создадим проект с именем hello

                        dcr new hello

На выходе получаем такую структуру

                        hello/
                        - src/
                        - - main.c
                        - dcr.toml

2. Сборка проекта

Заходим в папку проекта hello

                        cd hello

Собираем проект с флагом --release

                        dcr build --release

На выходе получаем такую структуру

                        hello/
                        - src/
                        - - main.c
                        - target/
                        - - release/
                        - - - main
                        - dcr.toml
                        - dcr.lock

3. Запуск проекта

Для запуска проекта можно использовать 2 способа

Первый способ(рекомендуемый)

                            dcr run --release

Эта команда соберет проект, если он не был собран и запустит его

Второй способ

                            ./target/release/main

Так мы запустим напрямую проект

Если у вас есть вопрос по какому то из шагов, посмотрите документацию
